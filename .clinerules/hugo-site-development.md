## Brief overview
这是针对当前 Hugo 静态站点项目的专属开发规则文件，适用于本项目的所有开发、修改与维护工作

## 项目环境
- 本项目为 Hugo 静态站点，工作目录位于 `/home/book/my-hugo-site`
- 使用标准 Hugo 项目目录结构
- 所有文件操作均基于当前工作目录执行
- 已支持的 CLI 工具：git, hugo, npm, go 等

## 开发工作流
- 修改前先读取对应文件内容确认当前状态
- 优先使用 `replace_in_file` 进行局部精准修改
- 仅在新建文件或全量覆盖时使用 `write_to_file`
- 每次工具调用后等待执行结果确认，再进行下一步操作
- 代码修改完成后需进行验证检查

## 语言与沟通规范
- 所有交互、代码注释、文档、提交信息必须使用 中文
- 回复保持直接、技术导向，避免客套用语
- 任务执行步骤清晰，使用 task_progress 清单跟踪进度

## 编码与文件规范
- 遵循已有项目的代码风格与缩进格式
- 修改配置文件时保持原有结构顺序
- Markdown 文件使用标准 Hugo 前置元数据格式
- 文件名使用连字符 `-` 分隔单词，不使用下划线

## Git 提交规范
- 严格遵循 Conventional Commits 格式：`<type>(<scope>): <subject>`
- 提交标题长度不超过 50 个字符
- 提交信息全部使用中文描述
- 支持类型：feat, fix, docs, style, refactor, perf, test

## 工具使用原则
- 优先使用内置工具而非手动执行命令
- 安全操作无需用户确认，危险操作需显式申请审批
- 每次消息只执行一个工具操作，逐步推进任务
- 任务完成后必须使用 attempt_completion 确认结果

## 更新说明
- 每当项目结构发生变化时，更新ReadMe的项目结构
- 使用list_files列出文件列表
- 使用树形目录，具体到每一个文件
