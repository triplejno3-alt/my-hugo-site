# Workflow: 站点日常维护

本工作流用于指导 Cline 协助用户进行 Hugo 站点的日常技术维护、性能优化和结构更新。

## 触发条件 (Triggers)
- 用户提出优化站点的需求（例如：“帮我优化一下 SEO” 或 “检查一下站点性能”）。
- 用户需要更新项目结构（例如：“更新 ReadMe 的目录树”）。
- 项目依赖或 Hugo 版本需要更新。

## 执行步骤 (Steps)

### 1. 状态评估
- **动作**：使用 `ask_followup_question` 询问需要进行的维护类型。
- **优化**：提供 `options` 供用户快速选择（如：性能优化、SEO 检查、同步目录树）。
- **动作**：运行 `hugo` 构建检查当前是否有报错或警告。
- **动作**：使用 `list_files` 查看项目结构，确认是否符合标准。

### 2. 结构与文档同步
- **动作**：根据 `.clinerules/hugo-site-development.md` 要求，更新 `README.md` 中的项目结构树。
- **规范**：保持树形目录具体到每一个文件。

### 3. SEO 与元数据检查
- **动作**：扫描 `content/` 目录下的所有文章，检查是否缺失必要的 Front Matter（如 `description`, `tags`）。
- **动作**：检查静态资源的压缩情况（如图片是否需要提示压缩）。

### 4. 依赖更新 (可选)
- **动作**：检查 `themes/` 或 `git modules` 是否有更新。
- **动作**：检查 `npm` 依赖（如果存在）是否有安全漏洞。

### 5. 验证与提交
- **动作**：运行构建验证所有修改。
- **规范**：使用 Conventional Commits 记录维护操作（例如：`chore: 更新项目目录结构` 或 `perf: 优化文章元数据以提升 SEO`）。

## 关联规则
- `.clinerules/hugo-site-development.md` (基础开发规范)
